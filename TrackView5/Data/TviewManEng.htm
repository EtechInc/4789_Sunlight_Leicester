INTRODUCTION

 

TrackVIEW General

 

TrackVIEW 5 is the latest generation of a Supertrack monorail visualization/management package.

 

Standard Features

 

         High-speed communications protocol typical screen update less than 0.5 seconds

 

         15 + Computer Workstations (Full Control)

 

         Multi language interface.

 

         99 Category Groups, Up-to 999999 Category codes, 999999 customer codes. The ability to add a 25-character name for display/reporting purposes (sometimes shortened to save space). Up-to 4 characters abbreviated name for displaying in the bags.

 

         When using 99 Categories or less the Category Group and Category databases can be combined simplifying the options for operators.

 

         Rounded squares represent bags on the system, with up-to 2 pieces of information displayed. This information can be configured by the customer to show up to 4 pieces of information, 2 by name/number and 2 by colour codes.

 

         The screen layouts are drawn as close as possible to the actual system layout. Position of the workstation is considered

 

         Bag details can be viewed by moving the mouse pointer over the bag. All known information then appears in a pop-up box

 

         4 password-protected access levels (Operator, Supervisor, Engineer and Programmer).

 

         Full editing of bag information

 

         20 step unloading program for each discharge point. Up to 99 bags can be taken on one program step. 1st, 2nd and 3rd Category Group choices. Option to use Automatic customer selection.

 

         Almost unlimited loading/unloading programs (hard disk space is the only limit) can be stored.

 

         Alarm system gives both text and visual notification.

 

         The ability to control system behaviour with simple onscreen interfaces.

 

         PLC machine management. Each element of the Supertrack system is controlled by a local sequence, and line counters. These sequences can be interlinked with each other. Engineers can view and adjust the sequences and counters via the TrackVIEW menu. This can then be used for diagnostic purposes.

Category/Category Group Database

 

Depending on the data requirements the system can be set in 2 ways.

 

1)    Category Database.

 

This is the simplest the database and should be used in most situation where a category number of 1-99 is used. All Supertrack parameter can then be assigned to the relevant category number.

 

2)    Category Group Database.

 

This is a more complex way of managing the database. A category group number (1-99) is used to assign the Supertrack parameter. This group number is then used to group the categorys (1-999999) so that they can be processed through the Supertrack system.

 

These Database options will be set at the commissioning stage by the system programmer as these settings can have implications on the PLC program.

PLC Counters

 

Some parts of the system use counting trips. These are used to determine whether a line is full, has space or a bag is going to the line.

 

 

 

 

 

On the above example, the counting trips are shown behind the last bag on each line. In reality they are positioned between the front and back trolley (wheels) of the last bag.

 

How they work:

 

         When the count is 2, the line is considered clear.

         When a bag is released to the line the count is set to 0 (bag expected).

         When the front trolley (wheel) hits the trip, the count is then set to 1 (line full).

         When the back trolley (wheel) hits the trip, the count is then set to 2 (line clear).

 

When a bag is sent to a nearly full line, only the front trolley will make the trip. Therefore, the count will stay at 1 indicating the line is full.

 

The counter control operates in the same way as the sequence control. It can be opened from both the Diags menu and by selecting a counter from the screen.

PLC Status

The status of each PLC module can be viewed in TrackView.

 

 

Not all modules are used in any system. Any unused modules are shown with a grey lamp. Modules that are reporting a status of OK are shown with a green lamp and Modules with an error are shown with a red lamp. If a module is reporting an error then the exact problem can only be found by looking at the indication LEDs on the module with a problem.

 

The weighing modules on the Loading Stations are not part of the normal Input and Output system and can be seen separately by selecting Loading Stations.

Bag Movement Alarms

 

Each PLC sequence can have alarms. Its activation is when something is incorrect for the sequence step or an action takes too long. It is designed to give simple to understand messages and a simple icon that flashes to represent the physical position of the fault.

 

The engineer can also look at the PLC sequence associated with the item and see what the PLC thinks is happening.
